Предназначен для просмотра информации о предупреждениях и связанных с ними объектах программы.

При выполнении синхронизации данных возможно возникновение проблемных ситуаций, требующих вмешательства пользователя.

Программа предоставляет возможность:

  • расследовать проблемы в обмене данными;
  • исправлять ошибки в исходных данных, препятствующие выгрузке;
  • под свою ответственность включить обмен проблемными данными.

С помощью кнопок панели навигации в списке можно:

  • Обновить - список не является динамическим, поэтому для получения новых предупреждений нажмите кнопку;
  • Открыть объект - быстрый переход к объекту программы - предмету предупреждения для исправления ошибок;
  • Скрыть из списка - не показывать это предупреждение. Кнопка выключает видимость предупреждения в списке, при этом само предупреждение остается в программе и к нему можно будет вернуться позже. Надо понимать, что проблема этим не решается, а игнорируется, откладывается на "потом".

Групповая обработка

С помощью кнопки Групповая обработка можно открыть выпадающее меню и выбрать нужную команду для быстрого решения проблем из нескольких однотипных предупреждений:

  • Проведение документов - после исправления ошибок несколько документов можно провести одновременно;
  • Групповое изменение реквизитов - можно заполнить или исправить реквизиты в нескольких объектах программы;
  • Коллизии - в списке коллизий с помощью кнопок панели навигации можно проверить правильность автоматического решения конфликта и выбрать нужную версию данных;
  • Запрет загрузки по дате - в списке с помощью кнопок панели навигации можно просмотреть обе версии данных и выбрать нужную.

Команды для работы с предупреждениями

В контекстном меню списка по правой кнопке мыши и в меню Еще можно найти дополнительные команды для работы со списком:

  • Показать скрытые в списке - включить видимость ранее скрытых предупреждений;
  • Показать в списке - с помощью этой команды скрытые предупреждения становятся активными, с ними можно работать;
  • Журнал регистрации в указанное время - показывает весь журнал регистрации без отборов относительно поля Дата возникновения предупреждения + - 1 секунда от события, можно просмотреть и те процессы, которые происходили параллельно синхронизации. Это позволит проанализировать причины возникновения предупреждения. 
  • С помощью ссылки в нижней части списка можно Перейти в журнал регистрации с отбором по текущей дате.

Отборы

Для удобства просмотра предупреждений предусмотрены отборы:

  • Период;
  • Синхронизация - с помощью флажка выберите синхронизацию, по которой нужно отработать предупреждения. При переходе из списка синхронизаций автоматически выбирается отбор по выбранной синхронизации;
  • Тип - выберите нужные типы с помощью флажков:

Ошибки проверки перед отправкой (XDTO)

  • Ошибки проверки сконвертированного объекта дополнительно дублируются в журнале регистрации и говорят о том, что при попытке отправить данные программа обнаружила пустые сведения, которые обязательно должны быть заполнены. 
  • Можно перейти в объект программы с помощью кнопки панели навигации списка или ссылки в предупреждении и заполнить недостающие реквизиты. Иногда объект может не отображаться по умолчанию. В этом случае следует обратиться к администратору программы.

Непроведенные документы

  • Для решения проблемы с проведением документа откройте карточку предупреждения, перейдите в документ по ссылке. 
  • Необходимо устранить недочеты, мешающие его проведению (это может быть, например, недостаточный остаток по товару), а после этого провести. При проведении предупреждение будет удалено из списка.

Незаполненные реквизиты

Необходимо открыть объект программы (с помощью кнопки панели навигации списка или по ссылке из карточки предупреждения), заполнить обязательные для заполнения поля, затем сохранить изменения. При записи исправлений предупреждение будет автоматически удалено.

Коллизии

  • При настроенной синхронизации данных между двумя программами возникают ситуации, когда одни и те же данные изменяются одновременно (в промежуток между выполнением сеанса синхронизации данных) в обеих программах. Это приводит к возникновению двух различающихся версий одинаковых данных в этих программах. Такая ситуация называется конфликтом изменения данных.
  • Такие конфликты разрешаются автоматически (программа выбирает одну из версий), однако можно изменить результат автоматического решения конфликта.
  • В карточке предупреждения подробно описывается суть проблемы, выводится ссылка на объект программы, по которому произошла коллизия. Можно перейти по ссылке и проконтролировать правильность заполнения реквизитов после автоматического разрешения конфликта.
  • С помощью кнопок в карточке предупреждения можно:
  • Сравнить две версии данных с помощью кнопки Показать отличия;
  • Для принятия решения, какую версию оставить, можно просмотреть обе версии. Для этого предусмотрены кнопки Показать версию в этой программе или Показать версию из другой программы.
  • После принятия решения о версии, которая должна быть сохранена, нажмите кнопку Скрыть из списка для сохранения результата автоматического разрешения конфликта или кнопку Пересмотреть результат для изменения конфликта на противоположный.

Непринятые по дате запрета

  • В этих предупреждениях выводятся сведения о данных, которые были отклонены из-за установленного запрета на загрузку данных прошлых периодов.
  • По ссылке или с помощью кнопки панели навигации можно открыть объект программы и посмотреть, правильно ли заполнены реквизиты в результате автоматического решения проблемы. 
  • Для сравнения отклоненных данных с текущими данными в программе нажмите кнопку Показать отличия (при наличии данных в текущей программе).
  • Просмотреть версии данных для сравнения можно с помощью кнопок Показать версию этой программы или Показать непринятую версию.
  • Для получения данных несмотря на установленный запрет загрузки данных можно нажать кнопку Принять отклоненную версию.

Ошибки кода обработчиков при отправке (получении)

  • Ошибки обработчиков выделяются в списке красным цветом. В карточке предупреждения обработчиков отправки/получения указывается, что были обнаружены ошибки в коде правил конвертации объектов.
  • Указываются пути решения проблемы:
    • обновиться на актуальную версию программы;
    • использовать актуальные правила конвертации.

Удаление устаревших предупреждений

  • С помощью ссылки в нижней части списка можно удалить устаревшие предупреждения.
  • При удалении предупреждений можно указать дополнительные параметры для отбора предупреждений:
    • Период;
    • Синхронизация;
    • Тип.
  • С помощью флажка Только скрытые записи, которые удовлетворяют условиям выше можно дополнить отбор и удалить те предупреждения, которые были помечены как скрытые. Внимание! 
    • Команда Скрыть из списка только отключает видимость предупреждений, а команда Удаление устаревших предупреждений физически удаляет предупреждения без возможности их восстановить.
  • Поэтому рекомендуется использовать эту команду ответственно, потому что удаление большей части предупреждений происходит автоматически при решении проблемы, а удаление предупреждений, связанных с коллизиями и датой запрета загрузки, приведет к удалению версий объектов.
