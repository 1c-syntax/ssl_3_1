Предназначена для создания резервной копии информационной базы.

Рекомендуется создавать резервные копии перед любой операцией в приложении, которая может необратимо изменить большие объемы данных, например, перед групповым изменением реквизитов объектов приложения. В этом случае необходимо выполнить резервное копирование в ручном режиме.

Открывается по команде Создание резервной копии из раздела Администрирование - Обслуживание - Резервное копирование и восстановление.

Ограничения

  • Внимание! Резервное копирование не поддерживается в случае, если пользователь Администратор, от лица которого запускается резервное копирование,имеет аутентификацию операционной системы.

Создание резервной копии

  • В верхней части окна выводится информация о том, когда последний раз выполнялось резервное копирование.
  • С помощью кнопки Выбрать в поле Каталог для сохранения резервной копии (обязательное поле для заполнения) укажите папку, в которую приложение сохранит резервную копию информационной базы. Рекомендуется выполнять резервное копирование на другой компьютер или внешний носитель.
  • Если в приложении имеются текущие сеансы, приложение выводит предупреждение об этом. Перейдите по ссылке, для того чтобы Посмотреть список активных сеансов. Работу пользователей в приложении необходимо завершить.
  • После установки необходимых параметров нажмите Сохранить резервную копию.

Процедура резервного копирования

  • Приложение попытается произвести резервное копирование в настоящий момент. При этом в окне выводится сообщение Проводится подготовка к созданию резервной копии и ссылка Активные пользователи, по которой можно открыть список подключений к приложению. Приложение предпримет попытку завершения работы всех пользователей.
  • После завершения работы всех пользователей приложение установит блокировку и выполнит резервное копирование.
  • Если приложение не сможет завершить работу всех пользователей, то она предложит выполнить резервное копирование при подключенных к приложению пользователях. С предложением приложения можно согласиться, если администратор уверен в том, что подключенные пользователи не вводят данные в приложение (например, если пользователи в конце рабочего дня забыли выйти из приложения). В противном случае необходимо завершить сеансы таких пользователей на их рабочих местах.
  • После успешного окончания резервного копирования приложение запускается снова, при этом выводится сообщение о выполненной процедуре резервного копирования, в котором указан путь к файлу с резервной копией.
  • Если приложению не удалось принудительно отключить сеансы, и администратором также не были предприняты какие-либо действия, то резервное копирование может не состояться. В этом случае приложение выводит предупреждение, что резервное копирование не было проведено, и список сеансов, которые не удалось завершить (например, Конфигуратор).

Запуск приложения в случае ошибок во время резервного копирования

  • Перед началом резервного копирования приложение выводит сообщение с рекомендациями по немедленному запуску.
