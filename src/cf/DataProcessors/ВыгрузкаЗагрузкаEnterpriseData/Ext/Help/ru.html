Выгрузка и загрузка данных EnterpriseData

Обработка предназначена для выгрузки и загрузки данных с использованием универсального формата EnterpriseData. Предназначена для использования в информационных базах, поддерживающих синхронизацию данных через универсальный формат.

Работа обработки, встроенной в конфигурацию

Обработка, поставляемая в составе конфигурации, работает только в режиме загрузки данных.

Для загрузки данных необходимо указать путь к файлу, содержащему данные для загрузки, и нажать кнопку Выполнить.

Для выгрузки данных, а также использования дополнительных возможностей следует использовать внешнюю версию данной обработки, входящую в состав поставки.

Упрощенный режим работы

Для выгрузки данных необходимо:

  1. Указать Путь к файлу выгрузки, в который будут выгружены данные
  2. Указать Версию формата, в которой необходимо выгрузить данные
  3. Настроить состав выгружаемых данных:
    1. Выбрать объекты метаданных
    2. При необходимости указать отбор по периоду для документов
    3. При необходимости настроить произвольные отборы для справочников и документов
  4. Выполненные настройки отборов можно сохранить в файл и использовать при следующих сеансах работы (кнопки Сохранить настройки и Восстановить настройки).

 

Для загрузки данных необходимо указать путь к файлу, содержащему данные для загрузки

Режим разработчика

Режим можно включить с помощью кнопки Еще. Предоставляет следующие дополнительные возможности:

  1. Выгрузка и загрузка данных из текста. В этом случае в качестве источника и получателя данных служит не файл, а текстовое поле расположенное на форме. Данный режим удобен, если данных немного, и результат выгрузки удобно перенести в обработку загрузки через буфер обмена.
  2. Использование внешних модулей обмена через универсальный формат. Для этих целей следует указать путь к внешней обработке, в модуле которой расположен внешний модуль менеджера обмена.
  3. Выгрузка данных, зарегистрированных на узле плана обмена. Для этих целей следует указать Узел обмена при выгрузке. В этом случае настройки состава выгружаемых данных и настройка отбора недоступны.
  4. В режиме разработчика администратору информационной базы становиться доступно переопределение каталога временных файлов при загрузке сообщения. Для этого необходимо использовать опцию Для временных файлов использовать каталог сообщений обмена данными. При включении опции в качестве каталога временных файлов будет использовать каталог, который указан в константе КаталогСообщенийОбменаДаннымиДляWindows или КаталогСообщенийОбменаДаннымиДляLinux. Администратор приложения самостоятельно контролирует заполненность константы, а так же доступность указанного каталога для пользователя, от лица которого будет выполнена загрузка данных.
