В случае если к другому приложению нет возможности прямого подключения по локальной сети или через Интернет (веб-сервисы), рекомендуется использовать другие каналы связи, например FTP-сервер. При этом приложения, между которыми настроена синхронизация данных, могут работать полностью автономно друг от друга и в разное время. Такой вариант подходит не только для синхронизации данных с приложениями, расположенными в других сетях (офисах), но и для приложений в одной сети.

При настройке синхронизации будет сформирован файл(ы) с настройками подключения (*.xml и *.json). В приложении-корреспонденте при выборе типа подключения, необходимо выбрать опцию Продолжение настройки, выбрать необходимый тип подключения и загрузить файл с настройками подключения

  • Введите Путь к каталогу и Порт компьютера для передачи данных.
  • Введите имя Пользователя (логин) и Пароль.
  • Укажите Максимальный размер файла, для того чтобы ограничить объем пересылаемого сообщения.
  • При необходимости включите флажок Пассивное соединение.
  • Перейдите по ссылке Параметры доступа в Интернет, для того чтобы настроить параметры прокси-сервера.
  • Включите флажок Сжимать отправляемые данные в архив , для того чтобы ограничить размер отправляемого сообщения.
  • Введите Пароль архива для сжатия и распаковки архива сообщения обмена. Подразумевается, что пароль сжатия в приложении-источнике и пароль распаковки в приложении-приемнике должны быть одинаковыми для пары обменивающихся приложений.
